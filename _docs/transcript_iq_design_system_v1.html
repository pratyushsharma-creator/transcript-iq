Why this direction

Transcript IQ is sold to analysts who run six LLM tabs in parallel and read at 2x speed. They expect intelligence products to feel like the tools they already love — Linear, Cursor, Anthropic Console, Claude. Bloomberg's aesthetic dies with Bloomberg. We skate where the puck is going.

"We don't dress information up. We make it dense, scannable, fluent. The brand isn't a logo or a colour — it's the velocity of every interaction. Considered. Fast. Native to the workflow."

Operating principle

Six pillars

Single-accent discipline

One brand colour — mint green — used across the entire system at varying intensities. Tier differentiation comes through gradient, glow, and tinted backgrounds — not a second chromatic. The brand reads as confident because it's not trying to do five things at once.

One colour, full range
Dark-mode-first, light-mode-equal

The default state is dark — because that's what your buyers' tools default to in 2026. Light mode is fully designed, never an afterthought. Both modes use the same single accent, calibrated for each context with separate glow and contrast values.

Native to their workflow
Mint green specifically

Distinctive in B2B intelligence (no major competitor uses green — AlphaSense blue, Bloomberg orange, FT pink, Pitchbook red, Tegus dark green but institutional). Modern AI-era brands have converged on this hue (Linear, Vercel accents, Cursor highlights). Coded as "go," "alpha," "growth" — exactly the signals your hedge fund and PE buyers want to see.

Category whitespace
Information density done well

Stripe-grade tables. Linear-grade lists. Cursor-grade panels. We assume the buyer has 3+ products to evaluate per session and wants information dense without cramped. 11px mono labels. 15px body. Tight line-heights on data, generous on prose. Hairline 1px borders, never 2px.

Built for power users
AI-era component primitives

Components that don't exist in heritage intelligence brands: source attribution chips with confidence bars, model attribution badges, command palette baked into navigation, live pulse indicators on fresh content, sparklines on metric cards, glass-morphism on overlays. The UI itself signals "this is AI-era."

Native to 2026
Geometric grotesque typography

Geist for everything. No serif. Serif is a heritage signal — wrong for a brand that wants to read as "now." Geist is what Vercel ships, what Anthropic uses internally, what every modern AI tool is converging on. Variable weight (300–700) carries the entire hierarchy. One typeface family covers all UI surfaces.

Why mint green, specifically

Category whitespace

AlphaSense (blue), Bloomberg (orange), FT (pink), Pitchbook (red), Capital IQ (blue), Tegus (dark green/black). Mint at #10B981 is unclaimed in B2B intelligence — instantly recognisable as Transcript IQ.

Distinctive
"Go / growth / alpha"

Green carries strong subconscious association with positive market direction, life, and forward motion. For a hedge fund or PE analyst — buyers literally optimising for alpha — this is the right semantic charge. Reads as confident, not cautious.

Buyer psychology
AI-era native

Linear has shifted toward green. Vercel uses green for success states. Cursor highlights are green-shifted. Modern AI tooling has converged on this hue family. Buyers see this colour and unconsciously code it as "modern intelligence tool."

Familiar to power users
Single-accent strength

Mint at full saturation reads premium without a second tier colour needed. Elite tier comes from gradient + glow + tint, not a separate amber or rose. The system stays unified. Fewer rules for users to learn, fewer rules for the team to maintain.

Discipline scales
Format-agnostic

#10B981 maintains chromatic identity at 11px and 64px. Survives both modes flawlessly. WCAG AA at body sizes. Tested against print, screen, dark mode, light mode, social cards, partner co-brand contexts.

Production-grade
Avoiding the consumer-fintech trap

Robinhood, Cash App, Acorns all use bright lime greens (#7FFF00 territory). We deliberately use mint #10B981 — blue-shifted, not yellow-shifted — paired with deep ink backgrounds and Geist typography. Reads premium institutional, never consumer.

Mitigated by execution

Typography system

One sans family. One mono family. No serif. Hierarchy comes from weight contrast and size — not from typeface family. Fewer fonts = faster page loads = the AI-era trade-off you actually want.

Usage rules

Discipline keeps the brand. Every rule below has a reason.

Colour rules

Do
Mint only for accent moments

Prices, primary CTAs, AI panels, links, single highlight per view.

Do
Both modes equally maintained

Every component must look intentional in both dark and light. No "compromise" mode.

Do
Hairline 1px borders

2px reads as utility/Bootstrap. 1px reads as considered.

Do
Glass-morphism on overlays only

Modals, dropdowns, popovers. Never on inline content surfaces.

Don't
Never two accent colours

Mint handles every accent moment. Tier through intensity, not new hue. No amber, no rose, no purple.

Don't
Never decorative gradients

Mint gradient on Elite tier and AI panels only. Never as page background or hero overlay.

Don't
Never neon/glow on body text

Glow on CTAs and pulses only. Glow on text reads as 2010s consumer app.

Don't
Never lime green

Reads as Robinhood / cash-app. Stay in mint family — blue-shifted, not yellow-shifted. Never #7FFF00 territory.

Typography rules

Do
Geist for everything

One sans family. Hierarchy from weight (300–700) and size, not from new families.

Do
Geist Mono for every number

Tabular figures via font-feature-settings. Slashed zero. Aligned vertically in lists.

Do
Tight letter-spacing on display

-0.04em at Display XL. -0.02em at body. Modern grotesque needs this.

Do
Sentence case everywhere

"Buy transcript", not "Buy Transcript" or "BUY TRANSCRIPT". Title case is dated, all caps is shouting. Mono labels can use uppercase eyebrow style.

Don't
Never use serif

Serif is a heritage signal. Wrong era for this brand. Including for blog post body.

Don't
Never below 13px body

Below 13 it stops being readable. Mono can go to 11 with letter-spacing.

Don't
Never weight 700+ on display

500–600 maximum. Heavier reads as advertising, not editorial.

Don't
Never system fonts in production

Always load Geist + Geist Mono explicitly via Google Fonts. System font fallback only during initial load.

Motion rules

Do
Use motion to communicate state

Pulse on live · fade on load · scale on press · slide on overlay.

Do
Cap at 320ms

Anything over 320ms feels slow. Page transitions snap, never cross-fade.

Don't
Never decorative animation

Floating particles, animated gradients, scroll-jacked sections — all banned.

Don't
Never spring physics on UI

Ease-out only. Springs read consumer-app, ease-out reads professional-tool.

Implementation rules · for Claude Code

Implementation rule
Use Tailwind utilities, not custom CSS

Every component should be expressible in Tailwind classes. Custom CSS only for animations, gradients, and CSS variables.

Implementation rule
Use shadcn/ui as base

Install shadcn components, then customise the color tokens to match this system. Don't roll your own primitive components.

Implementation rule
Always font-display: swap

Geist loads from Google Fonts CDN. Always set font-display: swap to avoid FOIT. System fallback is system-ui.

Implementation rule
Default to dark mode

Use next-themes with defaultTheme='dark'. Light mode is selectable but dark is the brand default.

Implementation rule
Never hardcode colours outside this file

Every colour reference goes through Tailwind tokens (mint-500, ink, accent) or CSS variables (var(--accent)). No raw hex in component files.

Implementation rule
Never use Inter, Roboto, or DM Sans

Even if Geist fails to load. Fall back to system-ui only. Other "modern" sans typefaces will subtly drift the aesthetic.

Loading the fonts

Use next/font in layout.tsx for optimal loading:

// app/layout.tsx import { Geist, Geist_Mono } from 'next/font/google' const geist = Geist({ subsets: ['latin'], weight: ['300', '400', '500', '600', '700'], variable: '--font-geist', display: 'swap', }) const geistMono = Geist_Mono({ subsets: ['latin'], weight: ['400', '500', '600'], variable: '--font-geist-mono', display: 'swap', }) export default function RootLayout({ children }) { return ( <html lang="en" className={`${geist.variable} ${geistMono.variable}`} data-theme="dark"> <body className="font-sans antialiased">{children}</body> </html> ) }

Hand-off note for Claude Code: This document is the source of truth for all visual decisions. When generating components, always reference these tokens. When in doubt about a colour, default to mint-500 for accents or ink for primary text. When in doubt about typography, default to Geist sans. When in doubt about layout, default to dark mode with hairline borders, generous whitespace, sentence case.
